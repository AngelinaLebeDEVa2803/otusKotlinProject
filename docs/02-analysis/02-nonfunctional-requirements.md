### Таблица нефункциональных требований

| ID | Категория | Требование | Метрика / Критерий приемки | Приоритет MVP | Комментарий для MVP |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **PERF-001** | Производительность | Скорость загрузки карты офиса и актуального статуса мест | ≤ 1.5 сек для 95% запросов при одновременной работе 20% сотрудников офиса | Must | Кэширование статичной карты этажа, асинхронная подгрузка статуса мест. |
| **PERF-002** | Производительность | Отклик системы при бронировании | Подтверждение бронирования ≤ 1 сек в 99% случаев | Must | Оптимизированные запросы к БД, быстрая обработка транзакций. |
| **REL-003** | Надежность | Доступность сервиса | **99.9% (три девятки)** в рабочее время (Пн–Пт, 8:00–20:00). Допускается ТО в ночное время или выходные. | Must | Для MVP достаточно развертывания в одном облачном регионе с daily backups. |
| **SEC-004** | Безопасность | Защита персональных данных | Полное соответствие ФЗ-152. Шифрование данных (PII) при передаче (TLS 1.3+) и хранении (AES-256). | Must | Обязательное требование для работы с данными сотрудников. |
| **SEC-005** | Безопасность | **Аутентификация и авторизация через Keycloak** | Реализация безопасного потока OAuth 2.1/OpenID Connect (Authorization Code Flow with PKCE).<br>1. **Время ответа Keycloak:** p95 ≤ 500 мс.<br>2. **Доступность Keycloak:** 99.95% (как критической инфраструктуры).<br>3. **Защита токенов:** `access_token` и `id_token` - JWT, подписанные алгоритмом RS256, срок жизни `access_token` ≤ 15 мин, `refresh_token` ≤ 24 ч.<br>4. **Соответствие стандартам:** OWASP ASVS V4.0 (требования V2, V3). | Must | Использование корпоративного экземпляра Keycloak. Настройка Confidential Client для бэкенда и Public Client с PKCE для фронтенда. Интеграция с корпоративной LDAP/AD провайдером. |
| **USAB-006** | Удобство | Адаптивность интерфейса | Корректное и удобное отображение на мобильных устройствах (≥90% от общего трафика). Карта офиса должна быть читаемой на экране 4.7". | Must | Mobile-first дизайн является критичным, так как бронирование часто происходит "на ходу". |
| **USAB-007** | Удобство | Интуитивность интерфейса | Новый пользователь (стажер) должен совершить первое бронирование без инструкций не более чем за 2 минуты. | Should | Проверка юзабилити-тестированием на фокус-группе. |
| **COMP-008** | Совместимость | Интеграция с календарями | Поддержка Google Calendar и Microsoft Outlook (Exchange Online). Создание и обновление событий должно происходить в течение ≤ 3 минут. | Must | Критичная интеграция для избежания двойного ввода данных и конфликтов. |
| **COMP-009** | Совместимость | Поддержка браузеров и ОС | **Браузеры:** Chrome, Firefox, Safari, Edge последних 2-х версий. **ОС:** iOS 15+, Android 10+. | Must | Отказ от поддержки устаревших браузеров (IE) для экономии ресурсов. |
| **SCAL-010** | Масштабируемость | Рост количества компаний-клиентов | Архитектура должна позволять подключать новые компании-клиенты (тенанты) без изменения кода или простоя сервиса. | Should | Заложение основ multi-tenant архитектуры с изоляцией данных на уровне БД. |
| **SCAL-011** | Масштабируемость | Обработка пиковых нагрузок | Система должна выдерживать пиковую нагрузку (понедельник, 9:00 утра) +300% к среднему показателю без деградации производительности. | Could | Для MVP можно использовать автоматическое масштабирование (autoscaling) в облачной среде. |
| **MAINT-012** | Ремонтопригодность | Ведение логов и мониторинг | Все критические действия (логин, бронь, отмена) логируются. MTTR (среднее время восстановления) ≤ 15 минут для критических инцидентов. | Should | Интеграция с системами мониторинга (Grafana, Prometheus) и оповещения (Telegram, Slack). |
| **LOC-013** | Локализация | Поддержка русского языка | Интерфейс, уведомления, документация и поддержка — на русском языке. Формат даты: `ДД.ММ.ГГГГ`, время: 24-часовой формат. | Must | Обязательное требование для B2B-рынка РФ. |
